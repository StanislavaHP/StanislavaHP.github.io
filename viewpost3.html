Lecture Notes 3: Methods C#


  1. Methods Definition – if a fragment of the code is used more than ones, it is good to define it as a method. These are code subroutines which in other languages are also known as functions or procedures.
  2. Why Use Methods?
    1. Better structured program and more readable code
    2. Avoid Duplicate Code
  3. Method Declaration
    1. Definition: method registration in the program, so it can be successfully identified
    2. Syntax: [Static] <return type> <method name>([<parameters list>]) {code}
    3. Location:
      1. Cannot be declared inside another method’s body (for example Main)
      2. Is declared between the opening and closing brackets of a class
    4. <return type>: The data type can be void (for no data returned) or any other data type.
    5. <method_name>: The Name of the method – should describe the function declared within the method.
    6. <parameters list>: The variable(s) which is(are) to be used by the method, if the method doesn’t use any variables we state ().
      1. Default Values: We can initialise variables with default values. In this case we don’t have to pass a value when calling the variable.
      2. Parameters and Arguments: Parameters are the elements in the parameter list, arguments are the values passed during method invocation to the parameters
  4. Method Implementation (Creation)
    1. Definition: Typing the code that resolves a particular task
    2. Operator Return:
      1. The returned variable could be in any data type under the one declared in the method. So for example if you declare double you can return int.
      2. Return would terminate the method and return the value specified after it. If you use return in a void method that would simply terminate the method.
      3. The operator return can be called from multiple places in the code of our method, but should be guaranteed that at least one of the operators return that we have used will be reached while executing the method.
    3. Variables:
      1. As with cycles the scope of any variables declared within the method are the {} of the method
      2. Variables inside a method are local
  5. Method Invocation
    1. Definition: The Process that invokes the already declared method. Call a method by MethodName (). Pass any variables (if required) within the brackets.
    2. Location:
      1. From the Main Program Method Main()
      2. From some other Method
      3. From its own body (recursion)
      4. If a method is called in the same class where it is declared and implemented it can be called at a line before the line at which it is declared.
    3. Passing variable
      1. What types can be passed: Expressions or Arguments compatible with the parameter type
      2. Effect: When an argument is passed to a method, the method takes a copy of the argument. If the declared parameter is of primitive type the usage of the argument does not change the argument itself (a copy of the value itself is being taken). If the declared parameter is of reference type, because the address (object reference) is the one being copied any changes would be applied to the object(value) itself.
      3. Named Arguments: in general you should follow the order in which the variables have been declared but you can switch the order if you identify the var name: MethodName (var1Name: x, var2Name: y)
  6. Method Result
    1. Initialise a variable with the result of a method
    2. Use a method in an expression
    3. Use the variable passed by a Method to pass to another Method
  7. Method Signature (Specification) – name of the method + parameter type + order in which the parameters are listed (the parameters names are irrelevant)
    1. Method Overloading (variations / overloads of the method) – two methods with the same name but different variable
    2. If two methods with the same name are defined, method1 (int, float) method1 (float,int) – when passing arguments we need to specify the type of the arguments, we can’t just use method1 (5,5) we can use method1 ((float)5, (short)5) or method1 (5f, 5).
  8. Program Execution – “The stack” stores information about the active subroutines (methods) of a computer program. It keeps track of the point to which each active subroutine should return control when it finished executing.
  9. Best Practice:
    1. Method
      1. Must resolve a distinct and well defined task
      2. The method resolves the task it is created for or returns an error.
      3. A method must have minimum depdency to the class in which the method is declared and to other methods and classes. This means that the method must do its job by using the data that is passed to it as parameters but not data that can be accessed from other places. Methods should not have side effects – change some global variables or print something on the console in the meantime.
      4. Should be short – avoid methods longer than 1 screen
    2. Method Name
      1. Identifies what the method does
      2. Starts with a capital Letter
      3. Each Word starts with a capital Letter
      4. Verb + Noun or Verb
    3. Parameters Name:
      1. First Lower, all other Words Capital
      2. Noun or Adj+Noun
      3. It is good to also include the metric
